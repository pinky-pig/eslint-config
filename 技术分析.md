# @arvinn/eslint-config 技术分析

## 架构概览
- **入口与导出**：`src/index.ts` 简单聚合导出。
- **配置片段**：`src/configs/*` 存放可复用的 Flat Config 片段（Prettier、Vue、TypeScript、UnoCSS 等）。
- **预设组合**：`src/presets.ts` 使用 `FlatConfigComposer` 组合各片段，生成 `arvinn()` 预设。
- **插件集中出口**：`src/plugins.ts` 统一 re-export 所有插件与解析器，供配置片段引用。
- **环境探测**：`src/env.ts` 提供 `hasVue()`、`hasUnocss()` 等，驱动预设的自动开关。
- **类型与生成物**：`src/types.ts` 定义通用类型，`src/typegen.ts` 由 eslint-typegen 生成规则类型。构建产物输出到 `dist/`。

## 核心流程
- 用户或本仓库的 `eslint.config.ts` 调用 `arvinn(options)`；`options` 控制哪些片段被启用。
- `arvinn()` 里把基础片段数组组装好，然后用 `FlatConfigComposer` 包装，返回 composer 实例。
- 使用方可以在返回的 composer 上链式调用 `.append()` / `.prepend()` 等，追加自己的 Flat Config。
- ESLint 解析 `eslint.config.ts` 时会展开 composer 内部的配置数组，最终把预设与追加配置一起应用。

## `arvinn()` 工作机制
`arvinn(options?: Options, ...userConfigs)` 定义于 `src/presets.ts`，返回 `FlatConfigComposer<Config, ConfigNames>` 实例（来自 `eslint-flat-config-utils`），因此可以链式 `.append()`、`.prepend()`。

- 基础配置数组起步：`presetBasic()`（JS/TS/Import/Unicorn/Node/JSdoc/Regexp/DeMorgan），`yml()`，`presetJsonc()`。
- 按选项/环境追加：
  - `prettier`（默认 true）→ `prettier()` 片段。
  - `vue`（默认 `hasVue()`）→ `vue()` 片段。
  - `markdown`（默认 true）→ `markdown()`。
  - `unocss`（默认 `hasUnocss()`）→ `unocss()`。
  - `command`（默认 true）→ `command()`。
  - `pnpm`（默认 false）→ `pnpm()`。
  - 始终：`specialCases()`。
- 最终执行：`new FlatConfigComposer(...configs, ...(userConfigs as any))`，把组装好的配置与用户传入的配置一起交给 composer。

### `options` 可用项（`arvinn({ ... })`）
- `vue`: 默认 `hasVue()`，开启 Vue 相关配置。
- `prettier`: 默认 `true`，开启 Prettier 集成。
- `markdown`: 默认 `true`，开启 Markdown 支持。
- `unocss`: 默认 `hasUnocss()`，开启 UnoCSS 规则。
- `command`: 默认 `true`，开启 eslint-plugin-command。
- `pnpm`: 默认 `false`，开启 pnpm 规则。
- `sortKeys`（未在默认分支中使用，可根据需要扩展）。
- 这些对应 `const { ... } = options` 里的解构及默认值，决定哪些 config 片段被 push 进最终数组。

### `.append()` 有什么不同
- `options` 只控制“预设”内置片段的启用/禁用，不影响序列之外的新增片段。
- `.append(...configs)` 则允许使用方在预设之后追加任意 Flat Config 块（如自定义规则、特定文件匹配配置等），不受 `options` 限制。
- 两者在实现上的区别：`options` 作用于 `arvinn()` 内部的 `configs` 组装；`.append()` 是 `FlatConfigComposer` 提供的链式 API，在实例化后追加更多配置。

#### 追加配置的两种等效写法
1) 作为 `arvinn` 的可变参数传入（直接跟在 `options` 后）：
```ts
export default arvinn(
  { vue: true, pnpm: true, unocss: false },
  {
    files: ['src/**/*.ts'],
    rules: { 'perfectionist/sort-objects': 'error' },
  },
)
```
2) 使用 `.append(...)` 链式追加：
```ts
export default arvinn({ vue: true, pnpm: true, unocss: false }).append({
  files: ['src/**/*.ts'],
  rules: { 'perfectionist/sort-objects': 'error' },
})
```
两种写法效果相同，都会把追加的配置放在预设片段之后，顺序按传入先后。

## `.append()` 为什么生效
`FlatConfigComposer` 内部维护一个配置序列，并提供 `.append(...configs)` 方法，把传入的 Flat Config 追加到序列。`eslint.config.ts` 导出 composer 时，ESLint 在解析时会展开 composer 的配置数组，因此 `.append()` 注入的配置会与默认预设一起生效。

本仓库示例（根目录 `eslint.config.ts`）：
```ts
import { arvinn } from './src/index'

export default arvinn({
  vue: true,
  pnpm: true,
  unocss: false,
}).append(
  {
    files: ['src/**/*.ts'],
    rules: { 'perfectionist/sort-objects': 'error' },
  },
  {
    files: ['**/*.md/*'],
    rules: { 'perfectionist/sort-named-imports': 'off' },
  },
)
```
两段 `.append()` 的配置会被加入 composer 序列，从而被 ESLint 采纳。

## Prettier 选项位置
- 库默认：`src/configs/prettier.ts` 中 `prettier/prettier` 规则的第二个参数可放 `prettierOptions`（已包含 `htmlWhitespaceSensitivity: 'ignore'`）。
- 使用方自定义：在项目的 `eslint.config.ts` 里覆写同名规则即可：
```ts
export default arvinn().append({
  rules: {
    'prettier/prettier': [
      'error',
      {
        htmlWhitespaceSensitivity: 'ignore',
        // 其他 Prettier 选项
      },
    ],
  },
})
```
